# Deploy to Cloudflare Pages (GitHub)

This project is configured for Cloudflare Pages using TanStack Start SSR.

## Cloudflare Pages settings

- **Framework preset:** None
- **Build command:** `bun run build`
- **Build output directory:** `dist`
- **Node version:** `22.16.0` or newer

## Environment variables

Add these in Cloudflare Pages → Settings → Environment variables for Production and Preview:

```env
NODE_VERSION=22.16.0

VITE_SUPABASE_URL=...
VITE_SUPABASE_PUBLISHABLE_KEY=...
VITE_SUPABASE_PROJECT_ID=...

SUPABASE_URL=...
SUPABASE_PUBLISHABLE_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...

LOVABLE_API_KEY=...
```

Use the same backend values as the Lovable project.

## Important

- Do **not** deploy this app to Vercel without a separate Vercel-specific migration.
- Do **not** set a Vercel output directory.
- Cloudflare must use the `dist` directory generated by the build.
